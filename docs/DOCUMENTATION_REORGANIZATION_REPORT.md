7. **Только важное** - убраны планы, отчеты, анализы (оставлены только гайды и архитектура)

## 📝 Правила работы с документацией

### Создание новых документов

1. **Текущая работа** → `work-in-progress/`
2. **После завершения** → перенести в соответствующий раздел:
   - Гайды → `guides/`
   - Архитектура → `architecture/`
   - Структуры → `structure/`
   - Флоу → `flows/`
   - Реализация → `implementation/features/`

### Обновление документов

- Добавлять метаданные в начало:
  ```markdown
  **Дата обновления:** YYYY-MM-DD
  **Статус:** ✅ Актуально / ⚠️ Частично устарело
  **Версия:** X.Y
  ```

### Архивирование

- Планы, отчеты, анализы → `archive/`
- Устаревшие документы → `archive/legacy/`
- Старые статусы → `archive/status/`
